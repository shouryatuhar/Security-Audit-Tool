# Contributing to HostSentinel

First off, thank you for considering contributing to HostSentinel! It's people like you that make HostSentinel such a great tool.

## Where do I go from here?

If you've noticed a bug or have a feature request, make one! It's generally best if you get confirmation of your bug or approval for your feature request this way before starting to code.

## Fork & create a branch

If this is something you think you can fix, then fork HostSentinel and create a branch with a descriptive name.

## Implement your fix or feature

At this point, you're ready to make your changes! Feel free to ask for help; everyone is a beginner at first.

## Code Review

Once your changes are ready, submit a Pull Request. We'll review your code, suggest any needed changes, and merge it when it's ready.

## Coding Style

- Use Python 3.8+ features and type hints.
- Write docstrings for all modules, classes, and methods.
- Avoid introducing unnecessary third-party dependencies.
